Text Formatting

for formatting text, I prefer normal (formal) rules, excepting the following:

  • no caps at the beginning of a sentence, unless it's a proper noun or pronoun.
  • no period at the end of a paragraph. if the sentence ends with other punctuation, I'll add it. this essentially just crosses over instant messaging and formal formatting.
  • other than that, it's just the same as formal English

for code, I generally follow these rules:

(I'll only talk about HTML and CSS here of course)

  • HTML
    • I always start out with the Emmet expansion of !, which looks like this: 
      <!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<meta name="viewport" content="width=device-width, initial-scale=1.0">
	<title>Document</title>
</head>
<body>
	|
</body>
</html>
    • I like using 4-space tabs because it looks nice and I can tell pretty easily if I'm nesting too much just by how indented a line is.
    • that's about it; I usually just use Prettier and it looks fine enough, though I don't like the 80-column limit and I'm fine if my lines don't wrap and go off the screen
  • CSS
    • I try to separate rule categories in a block. for example, here's this website's body block: 
      body {
	background-color: var(--black);
	color: var(--white);
	font-family: "Inter", sans-serif;

	margin-top: 2rem;
	margin-left: 25%;
	margin-right: 25%;
	margin-bottom: 2rem;
}
      as you can see, the first three lines are color and font (design), and the last four are margins (layout). I do this because it's easier to read and I can tell what's what at a glance

  • in general, it's best to not do spaghetti code and instead try to find an underlying issue. this makes your code so much easier to read, especially if you want/need to have someone look over it
